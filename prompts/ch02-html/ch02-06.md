> **Prompt**: What are ARIA attributes and when should beginners care about them?

think
**ARIA** (Accessible Rich Internet Applications) is a set of special attributes you add to HTML to help assistive technologies, like screen readers, understand what is happening on your page when standard HTML isn't enough.

If semantic HTML is a "labeled filing cabinet," ARIA is like adding **sticky notes** to the drawers to explain exactly what's inside or if a drawer is currently stuck.

---

## The Golden Rule: "No ARIA is better than Bad ARIA"
The first rule of ARIA is: **If you can use a native HTML element instead, do it.** * Use `<button>` instead of `<div role="button">`.
* Use `<nav>` instead of `<div role="navigation">`.

Native elements have accessibility built-in for free. ARIA is a "manual override," and if you use it incorrectly, you can actually make your site *less* accessible.

---

## When Should Beginners Care?
As a beginner, you don't need to memorize the hundreds of ARIA attributes. However, you should care about them in these **three specific scenarios**:

### 1. Labeling things that have no text (`aria-label`)
If you have a button that only contains an icon (like a "magnifying glass" for search or an "X" to close), a screen reader has nothing to read. 
```html
<button aria-label="Search">
  <i class="fa-search"></i>
</button>
```

### 2. Describing the "State" of an element (`aria-expanded`)
If you build a dropdown menu or an accordion, a blind user needs to know if the menu is currently open or closed.
* **`aria-expanded="false"`**: Tells the user "This can be opened, but it's closed right now."
* **`aria-expanded="true"`**: Tells the user "This is open and showing content."

### 3. Hiding decorative junk (`aria-hidden`)
Sometimes you have icons or images that are purely for looks and would only confuse a screen reader user (e.g., a "star" icon next to a rating).
```html
<span aria-hidden="true">★</span> 
<span>4.5 stars</span>
```

---

## The 3 Types of ARIA

| Category | What it does | Example |
| :--- | :--- | :--- |
| **Roles** | Defines **what** an element is. | `role="dialog"` (for a popup/modal) |
| **States** | Defines the **current condition**. | `aria-checked="true"` |
| **Properties**| Defines **relationships** or labels. | `aria-labelledby="heading-id"` |

## Summary Checklist for Beginners
* [ ] **Native First:** Did I check if an HTML5 tag (like `<main>` or `<header>`) exists for this?
* [ ] **Visible Text:** Does this button/link have actual text inside it? If not, use `aria-label`.
* [ ] **Don't Overdo It:** Don't add `role="button"` to a `<button>` tag. It’s redundant and can confuse some older screen readers.

> **Prompt**: Please show me 10 common beginner HTML mistakes and how to fix them.

Learning HTML is as much about unlearning "visual" habits as it is about learning tags. Browsers are very forgiving, so your code might *look* fine while being technically broken under the hood.

Here are 10 of the most common mistakes and the modern, accessible way to fix them.

---

### 1. The "Div-Soup" Trap
**Mistake:** Using `<div>` for every section, navigation bar, and footer.
**The Fix:** Use **Semantic HTML5** tags to give the structure meaning.
* **Bad:** `<div id="nav">...</div>`
* **Good:** `<nav>...</nav>` (Helps screen readers find the menu instantly).

### 2. Missing the `alt` Attribute on Images
**Mistake:** Leaving out `alt` text or using "image1.jpg".
**The Fix:** Provide a descriptive `alt` attribute for accessibility and SEO.
* **Bad:** `<img src="dog.jpg">`
* **Good:** `<img src="dog.jpg" alt="A golden retriever playing with a red ball.">`

### 3. Using Headings for Font Styling
**Mistake:** Using an `<h3>` just because you want the text to be smaller than an `<h2>`.
**The Fix:** Use headings for **structure only**. Use CSS for size.
* **Rule:** Headings should form an outline (h1 → h2 → h3). Never skip a level (e.g., h1 to h4).

### 4. Self-Closing Tag Confusion
**Mistake:** Trying to close tags that don't need it, or forgetting to close those that do.
**The Fix:** Remember that "Void elements" like `<img>`, `<br>`, and `<input>` do not have a closing tag (`</img>` is invalid).
* **Bad:** `<p>This is a paragraph. <br> New line. </p>` (The `<p>` must be closed, but `<br>` does not).

### 5. Incorrect Nesting (The "Tangled" Tags)
**Mistake:** Closing tags in the wrong order.
**The Fix:** Always close the **last tag opened** first.
* **Bad:** `<p><strong>Hello!</p></strong>`
* **Good:** `<p><strong>Hello!</strong></p>`



### 6. Using Inline Styles Instead of CSS
**Mistake:** Adding style attributes directly to HTML tags.
**The Fix:** Keep your structure (HTML) separate from your presentation (CSS).
* **Bad:** `<h1 style="color: blue; font-size: 20px;">Title</h1>`
* **Good:** Give it a class `<h1 class="main-title">` and style it in a `.css` file.

### 7. Creating Buttons with Links (and Vice Versa)
**Mistake:** Using an `<a>` tag for an action (like "Submit") or a `<button>` for navigation.
**The Fix:** * Use `<a>` if you are **going to a new URL**.
* Use `<button>` if you are **triggering a script** or submitting a form.

### 8. Forgetting the `<!DOCTYPE html>`
**Mistake:** Leaving out the very first line of the document.
**The Fix:** Always include `<!DOCTYPE html>`. Without it, browsers enter "Quirks Mode," which renders your CSS in a weird, outdated way.

### 9. Not Linking Labels to Inputs
**Mistake:** Placing text next to an input without a `<label>` tag.
**The Fix:** Use the `for` attribute to connect the label to the input `id`.
* **Bad:** `Name: <input type="text">`
* **Good:** `<label for="name">Name:</label> <input type="text" id="name">` (This makes the text clickable!)



### 10. Using "Click Here" for Link Text
**Mistake:** Making the link text generic.
**The Fix:** Make the link descriptive so users know where they are going.
* **Bad:** `To see our menu, <a href="/menu">click here</a>.`
* **Good:** `View our <a href="/menu">dinner menu</a>.`

---

## How to Catch These Automatically
You don't have to spot these by eye. You can use tools to do it for you:
1. **W3C Validator:** Paste your code into the [official validator](https://validator.w3.org/) to find syntax errors.
2. **VS Code Extensions:** Install **"HTMLHint"** or **"Prettier"** to highlight mistakes while you type.

# A cheat sheet table that summarizes these 10 rules

Here is a quick-reference "cheat sheet" you can bookmark or keep handy while you code. These 10 rules will help you avoid the most common pitfalls and ensure your HTML is professional, accessible, and search-engine friendly.

### HTML Best Practices Cheat Sheet

| # | The Mistake | The Modern Fix | Why It Matters |
| :--- | :--- | :--- | :--- |
| **1** | **Div-Soup** | Use `<header>`, `<nav>`, `<main>`, `<footer>` | Accessibility & SEO |
| **2** | **Missing `alt`** | Always add `alt="description"` to `<img>` | Screen readers & broken links |
| **3** | **Styling with `<h1>-<h6>`** | Use headings for hierarchy; CSS for size | Document structure |
| **4** | **Self-Closing `</img>`** | Void tags like `<img>` and `<br>` don't close | Valid syntax |
| **5** | **Tangled Tags** | Close in reverse order: `<b><i>...</i></b>` | Prevents layout "leaks" |
| **6** | **Inline Styles** | Use classes and external CSS files | Easier to maintain |
| **7** | **`<a>` vs `<button>`** | `<a>` for URLs; `<button>` for actions | Keyboard navigation |
| **8** | **Missing Doctype** | Start with `<!DOCTYPE html>` | Prevents "Quirks Mode" |
| **9** | **Unlinked Labels** | Use `<label for="ID">` with `<input id="ID">` | Larger clickable area |
| **10** | **"Click Here" links** | Use descriptive text like "View Pricing" | Accessibility links list |

### A "Gold Standard" Starter Template
If you start every project with this structure, you’ve already avoided 50% of the common beginner mistakes:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Accessible Page</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>

    <header>
        <nav aria-label="Main Navigation">
            <ul>
                <li><a href="/">Home</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section>
            <h1>Main Page Topic</h1>
            <p>Welcome to a well-structured page.</p>
            
            <form action="/submit" method="POST">
                <label for="user-name">Full Name:</label>
                <input type="text" id="user-name" name="name" required>
                <button type="submit">Submit Info</button>
            </form>
        </section>
    </main>

    <footer>
        <p>&copy; 2026 My Website</p>
    </footer>

</body>
</html>
```

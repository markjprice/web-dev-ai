> **Prompt:** Please show me three examples where `<div>` is appropriate and three where a semantic element would be better.

Choosing between a `<div>` and a semantic element usually comes down to whether the content has a specific **meaning** or if it is just there for **styling**.

A `<div>` is a "non-semantic" element; it tells the browser and assistive technologies (like screen readers) nothing about its content. Semantic elements, like `<nav>` or `<article>`, provide a roadmap for the page structure.

---

## When `<div>` is Appropriate
Use a `<div>` when the element's only purpose is visual or organizational, without adding any meaning to the document outline.

1.  **Layout Wrappers and Containers**
    If you need to center a block of content or create a Max-Width constraint for your CSS, a `<div>` is the perfect tool.
    ```html
    <div class="container">
      </div>
    ```

2.  **Grouping for CSS Styling (Flexbox/Grid)**
    When you need a parent element solely to apply `display: flex` or `display: grid` to align children, but that parent doesn't represent a distinct "section" of the page.
    ```html
    <div class="button-group">
      <button>Save</button>
      <button>Cancel</button>
    </div>
    ```

3.  **Decorative Elements**
    If you are creating visual flourishes like background shapes, dividers, or animated "cards" where the outer shell is just a hook for a background image or border.
    ```html
    <div class="decorative-blob"></div>
    ```

---

## When a Semantic Element is Better
Semantic elements help with SEO and accessibility by defining the "why" behind the code.

1.  **Navigation vs. Generic List**
    Instead of a `<div>` with a class of "nav", use `<nav>`. This tells screen readers exactly where the primary site links are located.
    * **Better:** `<nav><ul>...</ul></nav>`
    * **Avoid:** `<div class="menu"><ul>...</ul></div>`

2.  **Independent Content vs. Generic Section**
    If the content is a self-contained piece of information that could be distributed or reused (like a blog post, a comment, or a forum post), use `<article>`.
    * **Better:** `<article><h2>Post Title</h2><p>Content...</p></article>`
    * **Avoid:** `<div class="post"><h2>Post Title</h2><p>Content...</p></div>`

3.  **Secondary Information vs. Sidebar**
    For content that is related to the main topic but can be considered separate (like a glossary, related links, or an advertisement), use `<aside>`.
    * **Better:** `<aside><h4>Related Articles</h4>...</aside>`
    * **Avoid:** `<div class="sidebar"><h4>Related Articles</h4>...</div>`

---

### Summary Comparison Table

| Element | Role | Use Case |
| :--- | :--- | :--- |
| **`<div>`** | Generic Container | Centering content, applying CSS Grid/Flexbox, decorative wrappers. |
| **`<main>`** | Primary Content | Wrapping the unique, central content of a specific page. |
| **`<section>`** | Thematic Group | Grouping content that shares a specific heading or theme. |
| **`<footer>`** | Metadata/End | Copyright info, contact links, or "back to top" buttons. |

> **Prompt**: Please show me how to build a reusable SVG icon system using <symbol> and <use>.

A `<symbol>` and `<use>` system is the most efficient way to handle SVG icons on a modern website. Instead of pasting the same icon code 20 times, you define each icon **once** in a hidden "sprite sheet" and then "call" it wherever you need it.

This approach is better for performance because it keeps your HTML clean and allows for easy global updates.

---

## 1. The "Sprite Sheet" (Defining the Icons)
First, you create a single `<svg>` element at the top of your `<body>` (often hidden with CSS). Inside, you wrap each icon in a `<symbol>` tag. Each symbol needs a unique `id` and its own `viewBox`.

**The Code:**
```html
<svg style="display: none;">
  <symbol id="icon-check" viewBox="0 0 24 24">
    <polyline points="20 6 9 17 4 12" fill="none" stroke="currentColor" stroke-width="2"/>
  </symbol>

  <symbol id="icon-settings" viewBox="0 0 24 24">
    <circle cx="12" cy="12" r="3" fill="none" stroke="currentColor" stroke-width="2"/>
    <path d="M19.4 15a1.65 1.65 0 0 0 .33 1.82l.06.06a2 2 0 0 1 0 2.83 2 2 0 0 1-2.83 0l-.06-.06a1.65 1.65 0 0 0-1.82-.33 1.65 1.65 0 0 0-1 1.51V21a2 2 0 0 1-2 2 2 2 0 0 1-2-2v-.09A1.65 1.65 0 0 0 9 19.4a1.65 1.65 0 0 0-1.82.33l-.06.06a2 2 0 0 1-2.83 0 2 2 0 0 1 0-2.83l.06-.06a1.65 1.65 0 0 0 .33-1.82 1.65 1.65 0 0 0-1.51-1H3a2 2 0 0 1-2-2 2 2 0 0 1 2-2h.09A1.65 1.65 0 0 0 4.6 9a1.65 1.65 0 0 0-.33-1.82l-.06-.06a2 2 0 0 1 0-2.83 2 2 0 0 1 2.83 0l.06.06a1.65 1.65 0 0 0 1.82.33H9a1.65 1.65 0 0 0 1-1.51V3a2 2 0 0 1 2-2 2 2 0 0 1 2 2v.09a1.65 1.65 0 0 0 1 1.51 1.65 1.65 0 0 0 1.82-.33l.06-.06a2 2 0 0 1 2.83 0 2 2 0 0 1 0 2.83l-.06.06a1.65 1.65 0 0 0-.33 1.82V9a1.65 1.65 0 0 0 1.51 1H21a2 2 0 0 1 2 2 2 2 0 0 1-2 2h-.09a1.65 1.65 0 0 0-1.51 1z" fill="none" stroke="currentColor" stroke-width="2"/>
  </symbol>
</svg>
```

---

## 2. The "Instance" (Using the Icons)
To display an icon anywhere on your page, you use the `<use>` tag. You reference the `id` you created in the sprite sheet using `href="#id-name"`.

**The Code:**
```html
<svg class="icon"><use href="#icon-check"></use></svg>

<svg class="icon"><use href="#icon-settings"></use></svg>
```



---

## 3. Styling the System
Because we used `stroke="currentColor"` in the symbol definition, we can style the whole system with a few lines of CSS.

```css
.icon {
  width: 24px;
  height: 24px;
  color: #555; /* Default color */
  transition: color 0.2s;
}

.icon:hover {
  color: #007bff; /* Turns blue on hover */
}

/* You can also create size variations */
.icon-large {
  width: 48px;
  height: 48px;
}
```

---

## 4. Key Advantages of this System

* **DRY (Don't Repeat Yourself):** If you need to change the stroke width of your icons, you change it in **one** place (the symbol), and every icon on the site updates instantly.
* **Accessibility:** You can add a `<title>` tag inside the `<symbol>` to provide a description that screen readers will pick up for every instance.
* **Shadow DOM:** When you `<use>` a symbol, the browser creates a "clone" of the paths. This keeps your main DOM tree very small and easy to inspect.

---

## 5. Comparison: Inline vs. Symbol System

| Feature | Raw Inline SVG | Symbol + Use System |
| :--- | :--- | :--- |
| **Code Repetition** | High (Copies code everywhere) | Low (Single definition) |
| **HTML Cleanliness** | Messy (Walls of path data) | Very Clean (Small `<use>` tags) |
| **Global Updates** | Manual (Update every instance) | Automatic (Update the symbol) |
| **Styling** | Full control over every path | Limited (Styles apply to the whole clone) |

> **Pro-Tip:** If you have an icon that needs *different* colors for different parts (multi-color), the `<use>` tag has limitations because it clones the styles. In those specific cases, raw inline SVG is still the better choice.

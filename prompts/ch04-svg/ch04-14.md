> **Prompt**: Please explain how SVG filters (blur, shadow, glow) work with examples.

SVG filters are a powerful way to apply visual effects—like blurs, shadows, and glows—directly to vector elements. Unlike CSS box-shadows, SVG filters can follow the exact mathematical path of an object, no matter how complex the shape.

Filters work through a **Filter Primitive** system. You define a filter inside a `<defs>` tag and then "apply" it to an element using the `filter` attribute.

---

## 1. How the Filter Pipeline Works
SVG filters use a "building block" approach. Each tag (like `<feGaussianBlur>`) takes an input, processes it, and passes the result to the next stage.

* **SourceGraphic:** The original element (with its colors).
* **SourceAlpha:** Just the transparency/silhouette of the element (used for shadows).



---

## 2. The Gaussian Blur (`feGaussianBlur`)
This is the most basic filter. It spreads the pixels of an object to create a soft, out-of-focus look.

**The Code:**
```html
<svg width="200" height="200">
  <defs>
    <filter id="soft-blur">
      <feGaussianBlur in="SourceGraphic" stdDeviation="5" />
    </filter>
  </defs>
  <rect x="50" y="50" width="100" height="100" fill="green" filter="url(#soft-blur)" />
</svg>
```
* **`stdDeviation`**: The higher the number, the blurrier the object.

---

## 3. Creating a Drop Shadow (`feDropShadow`)
While you can build shadows manually using blurs and offsets, modern SVG supports a shorthand primitive that handles the "offset + blur + color" in one go.

**The Code:**
```html
<filter id="shadow">
  <feDropShadow dx="4" dy="4" stdDeviation="3" flood-color="black" flood-opacity="0.5"/>
</filter>
```
* **`dx` / `dy`**: How far the shadow is pushed horizontally and vertically.
* **`flood-color`**: The color of the shadow.

---

## 4. Creating an Outer Glow
To create a glow, you essentially create a blur of the object's color and then layer the original sharp object on top of it.

**The Code:**
```html
<filter id="glow" x="-50%" y="-50%" width="200%" height="200%">
  <feGaussianBlur in="SourceGraphic" stdDeviation="10" result="blurOut" />
  
  <feMerge>
    <feMergeNode in="blurOut" />
    <feMergeNode in="SourceGraphic" />
  </feMerge>
</filter>
```
> **Note on Filter Regions:** Notice the `x`, `y`, `width`, and `height` on the `<filter>` tag. By default, filters are clipped to 120% of the object. For big glows, you must expand this region so the glow isn't "cut off" at the edges.

## Common Mistakes
* **Forgetting IDs:** Filters are referenced by ID. If you have two filters with the same ID on one page, only the first one will work.
* **Performance:** Applying heavy blurs to many elements at once can slow down scrolling, especially on mobile. The browser has to "re-paint" the blur every time the element moves.
* **Clipping:** If your shadow or glow looks like it has a "flat edge," you need to increase the `x`, `y`, `width`, and `height` of the `<filter>` container.

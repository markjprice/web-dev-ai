> **Prompt**: Please explain how to make SVG icons theme-aware using currentColor and CSS variables.

Making SVG icons theme-aware allows them to automatically change color when you toggle between Dark Mode and Light Mode, or when you hover over a button. Instead of hard-coding hex codes like `#000000` inside your SVG, you link the SVG's internal properties to your CSS environment.

---

## 1. Using `currentColor`
The `currentColor` keyword is a CSS variable that represents the value of the element's `color` property. If you set `fill="currentColor"` inside your SVG, the icon will automatically take whatever text color is applied to its parent.

### **The SVG Code**
Remove the hard-coded hex fill and replace it with the keyword:
```html
<svg viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg">
  <circle cx="12" cy="12" r="10" stroke="currentColor" stroke-width="2" />
</svg>
```

### **The CSS Control**
Now, you control the icon entirely through standard CSS text properties:
```css
.icon-container {
  color: #333; /* Icon is now dark grey */
}

.icon-container:hover {
  color: #007bff; /* Icon turns blue on hover */
}

@media (prefers-color-scheme: dark) {
  .icon-container {
    color: #ffffff; /* Icon automatically turns white in Dark Mode */
  }
}
```



---

## 2. Using CSS Variables (Custom Properties)
If your icon uses multiple colors (e.g., a "primary" and a "secondary" accent), `currentColor` isn't enough because it only provides one value. For multi-tone icons, use **CSS Variables**.

### **The SVG Code**
Inline the variables directly into the attributes:
```html
<svg viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
  <rect x="2" y="2" width="20" height="20" fill="var(--icon-primary, #ccc)" />
  <circle cx="12" cy="12" r="5" fill="var(--icon-accent, #000)" />
</svg>
```

### **The CSS Theme Definition**
Define these variables in your global theme scope:
```css
:root {
  --icon-primary: #e0e0e0;
  --icon-accent: #1a73e8;
}

[data-theme="dark"] {
  --icon-primary: #3c4043;
  --icon-accent: #8ab4f8;
}
```

---

## 3. Comparison of Methods

| Method | Best For | Pros | Cons |
| :--- | :--- | :--- | :--- |
| **`currentColor`** | Monochromatic icons | Simplest to implement; works with standard `<a>` and `<button>` styles. | Only supports one color per icon. |
| **CSS Variables** | Multi-colored icons | Extremely flexible; can theme different parts of an icon independently. | Slightly more complex code; requires inline SVGs. |

---

## 4. Why this matters for Performance
By using these techniques, you avoid the "Icon Swap" anti-pattern. 
* **The Old Way:** Loading `icon-light.svg` and then swapping it for `icon-dark.svg` via JavaScript. This causes a flash of unstyled content (FOUT) and extra network requests.
* **The Modern Way:** You load **one** SVG file. The browser simply repaints the pixels when the CSS variables change, which is nearly instantaneous and uses zero extra bandwidth.



---

## 5. Important Limitation
Note that these techniques **only work for Inline SVGs**. If you load your SVG via an `<img>` tag (e.g., `<img src="icon.svg">`), the browser treats it as an encapsulated image, and it cannot "reach inside" to see your CSS variables or `currentColor`.

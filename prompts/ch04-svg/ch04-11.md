> **Prompt**: Please show me how to optimize an SVG file exported from Figma or Illustrator for use on the web.

When you export an SVG from professional design tools like Figma or Adobe Illustrator, the software often includes "junk" data intended for the editor, not the browser. This includes metadata, editor namespaces, hidden layers, and overly precise coordinates that increase file size and slow down your site.

Here is the step-by-step process to transform a design export into a high-performance web asset.

---

## 1. The Automated Optimization (SVGO)
The industry standard for SVG optimization is **SVGO** (SVG Optimizer). Most developers use the web-based version called **SVGOMG**.



### **Key Settings to Toggle:**
* **Precision:** Lower this to `1` or `2`. Designers often export with 6+ decimal places (e.g., `10.543218px`), but the human eye can't see that level of detail on a screen. Reducing precision can cut file size by 30%.
* **Remove Metadata:** Strips out Creator information and XML namespaces.
* **Remove Hidden Elements:** Deletes layers that are turned off but still exist in the code.
* **Collapse Groups:** Removes unnecessary `<g>` tags that don't have IDs or styles.

---

## 2. Manual Code Cleanup
Even after using a tool, you should check the raw code for these three common "Figma-isms":

### **A. Swap `width/height` for `viewBox`**
Figma often exports with fixed dimensions. To make the SVG truly responsive, ensure it has a `viewBox` and remove the fixed `width` and `height` (or set them to `100%` in your CSS).

**Before:** `<svg width="100" height="100">`
**After:** `<svg viewBox="0 0 100 100">`

### **B. Clean up IDs and Classes**
Design tools generate random IDs like `id="filter0_d_1_15"`. If you have multiple SVGs on one page with the same generated IDs, your filters or gradients will break because IDs must be unique. 
* **The Fix:** Rename them to something descriptive or use a tool to prefix them (e.g., `id="hero-logo-gradient"`).

### **C. Remove the `fill` from the SVG tag**
If you want to control the icon color via CSS `color: red;`, you need to change the internal paths to use `fill="currentColor"`.

---

## 3. Implementation Comparison

| Feature | Raw Export | Optimized SVG |
| :--- | :--- | :--- |
| **File Size** | Large (includes metadata) | Minimal (path data only) |
| **Readability** | Messy (nested groups) | Clean (flat structure) |
| **Responsiveness** | Often fixed `px` | Fluid `viewBox` |
| **Styling** | Hard-coded colors | `currentColor` support |

## 5. Final Checklist Before Production
1.  **Run through SVGOMG** (Precision set to 1).
2.  **Check for `viewBox`**.
3.  **Ensure `fill="currentColor"`** if the icon needs to change color.
4.  **Minify the HTML** if you are inlining the SVG directly into your code.

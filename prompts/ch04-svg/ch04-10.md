> **Prompt**: Please create a loading spinner using SVG and CSS animation.

To create a high-performance loading spinner, the best approach is to combine a simple **SVG circle** with **CSS Keyframe animations**. This method is superior to using an animated GIF because it scales perfectly to any size and allows you to change colors via CSS.



### 1. The SVG Structure
We use a `<circle>` element. The `stroke-dasharray` property is the "secret sauce"—it breaks the circle's border into a segment, which we then animate to create the spinning "gap" effect.

```html
<svg class="spinner" viewBox="0 0 50 50">
  <circle class="path" cx="25" cy="25" r="20" fill="none" stroke-width="5"></circle>
</svg>
```

---

### 2. The CSS Animation
We need two separate animations:
1.  **Rotate:** Spins the entire SVG container.
2.  **Dash:** Changes the length of the stroke so the spinner "grows" and "shrinks" as it moves.

```css
.spinner {
  animation: rotate 2s linear infinite;
  width: 50px;
  height: 50px;
}

.spinner .path {
  stroke: #4A90E2; /* Change spinner color here */
  stroke-linecap: round;
  animation: dash 1.5s ease-in-out infinite;
}

@keyframes rotate {
  100% {
    transform: rotate(360deg);
  }
}

@keyframes dash {
  0% {
    stroke-dasharray: 1, 150;
    stroke-dashoffset: 0;
  }
  50% {
    stroke-dasharray: 90, 150;
    stroke-dashoffset: -35;
  }
  100% {
    stroke-dasharray: 90, 150;
    stroke-dashoffset: -124;
  }
}
```

---

### 3. How it works
* **`viewBox="0 0 50 50"`**: Creates a coordinate system where the center is $(25, 25)$.
* **`stroke-dasharray`**: This defines the length of the "dashes" in the border. By animating this from a small value (`1`) to a larger value (`90`), the spinner looks like it's drawing itself.
* **`stroke-linecap: round`**: Gives the ends of the spinner a soft, rounded look instead of a harsh square edge.
* **`transform: rotate`**: Ensures the entire shape is constantly turning, preventing the "growth" animation from looking static.

---

### Why use this over a "CSS-only" Div spinner?
While you can make a spinner using a `div` with a `border-radius: 50%`, the SVG version is better for **Accessibility** and **Performance**.
* **GPU Acceleration**: Browsers optimize SVG path animations very efficiently.
* **Semantic**: You can add a `<title>Loading...</title>` inside the SVG so screen readers know exactly what is happening.

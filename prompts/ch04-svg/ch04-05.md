> **Prompt**: Give a step-by-step breakdown of how an SVG path is interpreted by the browser.

An SVG `<path>` is the most powerful element in the SVG arsenal. Unlike a simple `<rect>` or `<circle>`, a path is a series of commands that tell the browser's rendering engine exactly how to move a "virtual pen" across a 2D coordinate system.

The browser interprets these paths in a specific, linear sequence. Here is the step-by-step breakdown:

---

## 1. Attribute Parsing: The "d" String
The browser first looks for the `d` attribute (short for "data"). This attribute is a long string containing **commands** (letters) and **coordinates** (numbers). 

The browser reads this string from left to right, identifying two types of commands:
* **Uppercase (e.g., M, L):** Absolute coordinates (the exact point on the grid).
* **Lowercase (e.g., m, l):** Relative coordinates (move X units from the current position).



---

## 2. Command Execution: The Virtual Pen
The browser executes each command one by one, similar to how a plotter or a 3D printer works.

### Step A: The Initial Move (`M`)
Every path must start with a "Move To" command. 
* **Browser Action:** The browser lifts the virtual pen and places it at a specific $(x, y)$ coordinate without drawing anything.
* **Example:** `M 10 10` (Go to point 10,10).

### Step B: Drawing Lines (`L`, `H`, `V`)
Once the pen is down, the browser looks for line commands.
* **L (Line To):** Draws a straight line to $(x, y)$.
* **H / V:** Draws a strictly horizontal or vertical line.
* **Browser Action:** It calculates the mathematical vector between the current point and the new point and "paints" it.

### Step C: Calculating Curves (`C`, `S`, `Q`, `T`)
This is the most complex step. For a Bézier curve (`C`), the browser doesn't just draw a line; it uses **control points** to calculate a trajectory.
* **Browser Action:** It uses a mathematical formula (De Casteljau's algorithm) to determine the curve's arc based on how the control points "pull" the line toward them.



### Step D: The Closing Command (`Z`)
If the browser sees a `Z`, it draws a straight line from the current position back to the very first `M` coordinate of that path.

---

## 3. Applying the "Stroke" and "Fill"
Once the mathematical "skeleton" of the path is calculated, the browser's painting engine takes over:

* **Stroke:** The browser applies the `stroke-width` and `stroke-color` centered on the mathematical path.
* **Fill:** The browser determines what is "inside" the shape. If the path is complex or self-intersecting, it uses the **Fill Rule** (`nonzero` or `evenodd`) to decide which areas to color in.

---

## 4. Rasterization (The Final Step)
Finally, because your monitor is made of pixels and an SVG is made of vectors, the browser must **rasterize** the path.

* **Anti-aliasing:** The browser calculates sub-pixel values to ensure that diagonal lines and curves look smooth rather than "jagged."
* **GPU Hand-off:** If you are animating the path using CSS transforms, the browser may hand this rasterized "layer" to the GPU to move it efficiently.

---

## Real-World Example Breakdown
`d="M 10 80 L 40 10 L 70 80 Z"`

1.  **M 10 80:** Browser puts the pen at (10, 80).
2.  **L 40 10:** Browser draws a line from (10, 80) up to (40, 10).
3.  **L 70 80:** Browser draws a line from (40, 10) down to (70, 80).
4.  **Z:** Browser draws a final line from (70, 80) back to (10, 80) to close the triangle.

```svg
<svg width="100" height="100" viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <path d="M 10 80 L 40 10 L 70 80 Z" fill="orange" stroke="black" stroke-width="2" />
</svg>
```

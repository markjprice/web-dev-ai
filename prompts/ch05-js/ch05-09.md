> **Prompt**: What is event bubbling and capturing in JavaScript? Show a simple visual explanation.

In JavaScript, **Event Delegation** (the way events move through the DOM) is often compared to a "U-turn." When you click an element, the event doesn't just happen on that single spot; it travels from the root of the document down to the target and then back up again.

---

## 1. The Three Phases
Standard DOM events happen in three distinct phases:

1.  **Capturing Phase:** The event starts at the `Window` and `Document`, then moves down through every ancestor until it reaches the target element.
2.  **Target Phase:** The event reaches the element you actually clicked.
3.  **Bubbling Phase:** The event "bubbles" back up from the target, through the ancestors, all the way back to the `Window`.



---

## 2. Event Bubbling (The Default)
Bubbling is the most common behavior. If you click a button inside a `<div>`, the click event triggers on the button first, then the `<div>`, then the `<body>`, and so on.

**Why it's useful:** It allows for **Event Delegation**. You can put a single event listener on a parent `<ul>` to catch clicks on any of its `<li>` children, even if those children are added to the DOM later.

```javascript
// By default, listeners trigger during Bubbling
document.getElementById('child').addEventListener('click', () => {
  console.log('Child clicked!');
});

document.getElementById('parent').addEventListener('click', () => {
  console.log('Parent caught the bubble!');
});
```

---

## 3. Event Capturing (The "Trickle")
Capturing is the opposite. The event is caught by the parent first before it reaches the child. You rarely use this in everyday coding, but it is useful if you want to intercept an event before it reaches its intended target.

To use capturing, you must pass `true` (or an options object) as the third argument to `addEventListener`.

```javascript
// This will trigger BEFORE the child's click event
document.getElementById('parent').addEventListener('click', () => {
  console.log('Parent caught it during Capturing!');
}, true); 
```

---

## 4. Visual Comparison

| Feature | Event Bubbling | Event Capturing |
| :--- | :--- | :--- |
| **Direction** | Bottom $\rightarrow$ Up (Target to Root) | Top $\rightarrow$ Down (Root to Target) |
| **Default?** | **Yes** | No |
| **Analogy** | A bubble rising in a glass of water. | A stone dropping through water. |
| **Stop it?** | `event.stopPropagation()` | `event.stopPropagation()` |

---

## 5. How to Stop the Flow
Sometimes you want an event to happen on a button but **not** trigger the parent's click handler. You can use `event.stopPropagation()` to "break" the chain.

```javascript
button.addEventListener('click', (event) => {
  event.stopPropagation(); // The "U-turn" stops here; it won't bubble up.
  console.log('Only the button handled this.');
});
```

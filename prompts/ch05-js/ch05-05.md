> **Prompt**: What are the most useful console methods besides console.log?

While `console.log()` is the bread and butter of debugging, the Console API has several specialized tools that can save you significant time, especially when dealing with complex data structures or performance issues.

---

## 1. `console.table()`
This is arguably the most useful method for modern web development. If you have an array of objects or a large JSON response, `console.table()` renders the data as a sortable, readable table.

```javascript
const users = [
  { id: 1, name: 'Alice', role: 'Admin' },
  { id: 2, name: 'Bob', role: 'User' }
];
console.table(users);
```

## 2. `console.dir()`
While `console.log()` often shows you an HTML-like representation of a DOM element, `console.dir()` displays a functional, interactive list of all the properties of that JavaScript object. It is the best way to inspect the "true" state of a DOM node.

## 3. Grouping Content (`group` & `groupEnd`)
If you are logging multiple related items (like data inside a loop), the console can get cluttered. Use these to create collapsible sections:

```javascript
console.group('Fetching User Data');
console.log('Request sent...');
console.warn('Delayed response');
console.groupEnd();
```

## 4. Performance Timing (`time` & `timeEnd`)
Instead of manually calculating `Date.now()` differences, use the built-in timer. This is perfect for identifying "bottlenecks" in your code execution.

```javascript
console.time('loop-timer');
for (let i = 0; i < 1000000; i++) { /* complex math */ }
console.timeEnd('loop-timer'); // Outputs: loop-timer: 4.23ms
```

## 5. `console.assert()`
This allows for conditional logging. The message is only printed if the first argument is **false**. It helps keep your console clean by only reporting when something actually goes wrong.

```javascript
const value = 10;
console.assert(value === 20, 'Value is not 20!'); // This will log the error.
```

---

## Comparison Summary

| Method | Best Use Case |
| :--- | :--- |
| `console.warn()` / `error()` | Highlighting issues with specific color-coding and stack traces. |
| `console.count()` | Tracking how many times a function or component re-renders. |
| `console.trace()` | Finding the exact path (stack trace) that led to a specific line of code. |
| `console.clear()` | Cleaning the slate when the logs become overwhelming. |

> **Prompt**: What is the call stack in JavaScript and how does it relate to function execution?

The **Call Stack** is a fundamental mechanism that the JavaScript engine uses to keep track of its place in a script that calls multiple functions. Since JavaScript is **single-threaded** (it can only do one thing at a time), the stack is the "to-do list" that tells the engine which function is currently running and which ones should resume once the current one finishes.

---

## 1. How the Stack Works: LIFO
The Call Stack operates on a **Last-In, First-Out (LIFO)** principle. Think of it like a physical stack of cafeteria trays: 
* When you call a function, a new "frame" is pushed onto the **top** of the stack.
* When the function finishes, its frame is popped off the top.
* The engine then resumes the function directly underneath it.



---

## 2. A Concrete Example
Consider this code:

```javascript
function greet() {
  sayHi();
  console.log("Back in greet");
}

function sayHi() {
  console.log("Hi!");
}

greet();
```

### The Execution Steps:
1.  **Global Execution Context:** The script starts. The "Global" frame is at the bottom of the stack.
2.  **`greet()` is called:** A frame for `greet` is pushed onto the stack.
3.  **`sayHi()` is called:** Inside `greet`, the engine hits `sayHi()`. A new frame is pushed on top of `greet`.
4.  **`sayHi()` finishes:** After logging "Hi!", the function is done. Its frame is **popped off**. The engine looks at the stack and sees `greet` is back on top.
5.  **`greet()` finishes:** It logs "Back in greet" and is then **popped off**.
6.  **Empty Stack:** The script is finished (or returns to the Global context).

---

## 3. The Call Stack and Scope
The stack is directly related to **Scope**. Every time a function is pushed onto the stack, a new **Execution Context** is created. This context contains:
* The function's local variables.
* The arguments passed to it.
* A reference to its outer environment (the Scope Chain).

Because the engine only looks at the top of the stack, it can only "see" variables in the current execution context or those further down the chain.

---

## 4. When Things Go Wrong: Stack Overflow
The Call Stack has a finite size. If you call a function that calls itself (recursion) without an exit condition, the stack will keep growing until it runs out of memory.

```javascript
function recurse() {
  recurse(); // No exit condition!
}
recurse();
```
This results in the famous **"Maximum call stack size exceeded"** error. The engine essentially says, "I have too many trays and the stack is hitting the ceiling!"

---

## 5. Summary Table

| Feature | Description |
| :--- | :--- |
| **Mechanism** | Last-In, First-Out (LIFO) |
| **Push** | Occurs when a function is invoked/called. |
| **Pop** | Occurs when a function reaches `return` or the end of its block. |
| **Purpose** | To track the "Execution Context" and return address of functions. |
| **Single Thread** | Only the function at the very top of the stack is being executed. |

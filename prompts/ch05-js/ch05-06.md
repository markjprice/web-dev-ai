> **Prompt**: Please explain JavaScript scope and the difference between global, function, and block scope with examples.

**Scope** in JavaScript determines the accessibility (visibility) of variables. Think of it as a set of rules that defines where a variable can be used and where it is hidden.

In modern JavaScript, there are three primary types of scope: **Global**, **Function**, and **Block**.

---

## 1. Global Scope
Variables declared outside of any function or curly braces `{}` are in the global scope. They are accessible from **anywhere** in your code—inside functions, loops, or blocks.

* **Risk:** Global variables can lead to "namespace pollution," where different parts of your program accidentally overwrite the same variable name.

```javascript
const globalVar = "I am everywhere";

function checkScope() {
  console.log(globalVar); // Accessible here
}

checkScope();
console.log(globalVar); // Accessible here too
```

---

## 2. Function Scope
When you declare a variable inside a function using `var`, `let`, or `const`, that variable is local to the function. It cannot be accessed from outside.

* **Behavior:** Each function call creates a new "bubble" of scope.

```javascript
function hello() {
  const message = "Hello from inside!";
  console.log(message); // Works
}

hello();
console.log(message); // ReferenceError: message is not defined
```

---

## 3. Block Scope
Introduced in ES6 (2015), block scope applies to any code inside curly braces `{}`—such as `if` statements, `for` loops, or even standalone blocks. 

**Crucial Note:** Only variables declared with `let` and `const` are block-scoped. Variables declared with `var` **ignore** block scope and "leak" out to the nearest function or global scope.



### The Difference: `let` vs `var`
```javascript
if (true) {
  var leaked = "I am a var, I leak!";
  let trapped = "I am a let, I stay here.";
}

console.log(leaked);  // "I am a var, I leak!"
console.log(trapped); // ReferenceError: trapped is not defined
```

---

## Comparison Summary

| Scope Type | Defined By | Keywords | Accessibility |
| :--- | :--- | :--- | :--- |
| **Global** | Top-level of script | `var`, `let`, `const` | Everywhere in the script. |
| **Function** | Inside a `function` | `var`, `let`, `const` | Only inside that specific function. |
| **Block** | Inside `{ }` | `let`, `const` | Only inside that specific block. |

### Lexical Scope (The "Chain")
JavaScript uses **Lexical Scoping**, meaning the engine looks for a variable starting from the inner-most scope and moves outward. This is often called the **Scope Chain**. If it can't find the variable in the current block, it checks the function; if not there, it checks the global scope. If it still isn't found, you get a `ReferenceError`.

```javascript
const name = "Gemini";

function outer() {
  // If 'name' wasn't here, it would look at the Global Scope
  function inner() {
    console.log(name); // Looks up the chain to find "Gemini"
  }
  inner();
}
```

> **Prompt**: What is CSS specificity and how is it calculated step by step?

Specificity is the weight applied to a given CSS declaration. It is determined by the types of selectors used in the matching rule. When multiple rules target the same element, the browser uses a four-part calculation to determine which one "wins."

Think of specificity as a score represented by four numbers in separate columns: **(0, 0, 0, 0)**.



---

## The Calculation Levels (Step-by-Step)

To calculate the score, you look at the selector from left to right, adding 1 to the corresponding column for every instance of that selector type.

### 1. Inline Styles (1, 0, 0, 0)
Styles applied directly to an element in the HTML override any styles in a CSS file.
* **Example:** `<h1 style="color: blue;">`
* **Score:** **(1, 0, 0, 0)**

### 2. ID Selectors (0, 1, 0, 0)
IDs are unique identifiers. They are very powerful and will override classes or elements.
* **Example:** `#main-header`
* **Score:** **(0, 1, 0, 0)**

### 3. Classes, Attributes, and Pseudo-classes (0, 0, 1, 0)
This category includes standard classes, attribute selectors (like `[type="text"]`), and pseudo-classes (like `:hover`).
* **Example:** `.nav-item` or `:first-child`
* **Score:** **(0, 0, 1, 0)**

### 4. Elements and Pseudo-elements (0, 0, 0, 1)
The least specific tier. This includes standard HTML tags and pseudo-elements (like `::before`).
* **Example:** `div`, `p`, `section`
* **Score:** **(0, 0, 0, 1)**

---

## Real-World Scoring Examples

Let's see how these scores stack up against each other.

| Selector | Calculation | Final Score |
| :--- | :--- | :--- |
| `h1` | 1 Element | **(0, 0, 0, 1)** |
| `p.caption` | 1 Element + 1 Class | **(0, 0, 1, 1)** |
| `.header .title` | 2 Classes | **(0, 0, 2, 0)** |
| `#nav .link:hover` | 1 ID + 1 Class + 1 Pseudo-class | **(0, 1, 2, 0)** |
| `body #container ul li` | 1 ID + 3 Elements | **(0, 1, 0, 3)** |

---

## Important Rules of "Combat"

* **Higher Columns Trump Lower Ones:** A single ID `(0, 1, 0, 0)` beats any number of classes. You could have 1,000 classes, and the ID would still win.
* **The Universal Selector (`*`) has no weight:** It has a score of **(0, 0, 0, 0)**.
* **`!important` is the "Joker":** It isn't part of the specificity calculation, but it acts as a manual override. If a property has `!important`, it wins regardless of the specificity score.
* **The Tie-Breaker:** If two selectors have the exact same score (e.g., both are `(0, 0, 1, 0)`), the **last one defined** in the CSS file wins.

> **Prompt**: What are common mistakes that break responsive layouts?

Creating a responsive layout is as much about what you **don't** do as what you do. Most "broken" layouts—where content overflows the screen or elements overlap—stem from a few specific habits.

---

## 1. Using Fixed Widths
The most common mistake is using absolute units like `px` for large containers. If you set a `div` to `width: 1000px;`, it will look great on a desktop but will force a horizontal scrollbar on a smartphone.

* **The Mistake:** `width: 1200px;`
* **The Fix:** Use `max-width: 1200px;` combined with `width: 100%;`. This allows the element to be smaller than 1200px on small screens but stops it from stretching forever on large ones.

---

## 2. Neglecting the Viewport Meta Tag
If you forget the viewport meta tag in your HTML `<head>`, mobile browsers will assume you've built a desktop-only site. They will "zoom out" to fit the whole 1000px+ layout into the tiny screen, making your text microscopic.

* **The Fix:** Always include this line in your HTML:
    `<meta name="viewport" content="width=device-width, initial-scale=1.0">`

---

## 3. The "Width 100% + Padding" Explosion
As we touched on with the Box Model, adding padding to a `width: 100%` element expands it *beyond* the edges of the screen unless you change the sizing math.



* **The Mistake:** Default `box-sizing: content-box;`.
* **The Fix:** Apply `box-sizing: border-box;` globally at the top of your CSS file:
    ```css
    * {
      box-sizing: border-box;
    }
    ```

---

## 4. Hard-Coding Heights
Setting a fixed `height` on a container is a recipe for disaster. When the screen gets narrower, text wraps. If the text wraps, it needs more vertical space. If the container height is fixed, the text will "overflow" out of the box and overlap the elements below it.

* **The Mistake:** `height: 300px;`
* **The Fix:** Use `min-height: 300px;` or, better yet, let the content define the height naturally with padding.

---

## 5. Designing "Desktop-First" Only
While not a technical bug, designing for a large screen and then trying to "shrink" it down often leads to cluttered mobile views. 

* **The Fix:** Try **Mobile-First** design. Write your base CSS for the smallest screen (one column, simple layout) and use **Media Queries** to add complexity as the screen gets wider.



---

## 6. Large Images Breaking the Grid
By default, an image will display at its full resolution. If a user uploads a 4000px wide photo, it will blast through your 400px container.

* **The Fix:** Add a responsive image reset to your stylesheet:
    ```css
    img {
      max-width: 100%;
      height: auto;
    }
    ```

---

## 7. Using Too Many "Magic Numbers"
"Magic numbers" are random values like `margin-left: 173px;` used to "hack" an element into place. These almost always break when the screen size changes even slightly.

* **The Fix:** Rely on Flexbox, Grid, or percentages to handle alignment automatically.

> **Prompt**: Please explain inheritance in CSS and which properties inherit by default.

In CSS, **Inheritance** is the mechanism where certain properties are passed down from a parent element to its children. It’s the reason why, if you set the font color on the `<body>` tag, all the paragraphs and lists inside it automatically turn that color without you having to style them individually.

---

## How Inheritance Works
When a property is inherited, the browser checks the element for a specific value. If no value is defined for that element, the browser looks at the parent, then the grandparent, and so on, until it reaches the root (`<html>`).



### The "Default" Behavior
Not every CSS property inherits. If they did, things would get messy very quickly. Imagine if you gave a `<div>` a border, and suddenly every single link, span, and image inside that div also had a border. It would be a layout nightmare!

---

## 1. Properties That INHERIT by Default
Generally, properties related to **text and typography** inherit. These are things you typically want to remain consistent throughout a section of a page.

* **Color:** `color`
* **Font Properties:** `font-family`, `font-size`, `font-weight`, `line-height`, `letter-spacing`
* **Text Alignment:** `text-align`, `text-indent`, `text-transform`
* **List Styles:** `list-style-type`, `list-style-position`
* **Visibility:** `visibility`

---

## 2. Properties That DO NOT Inherit
Properties related to the **Box Model** and **Layout** do not inherit. These define the physical structure of a specific container.

* **Box Model:** `width`, `height`, `margin`, `padding`, `border`
* **Backgrounds:** `background-color`, `background-image`
* **Positioning:** `top`, `right`, `bottom`, `left`, `position`, `z-index`
* **Display:** `display`, `flex`, `grid`

---

## 3. Forcing Inheritance
Sometimes you *want* a non-inheriting property to behave like an inheriting one. You can use the `inherit` keyword to do this.

**The Code:**
```css
.parent {
  border: 2px solid black;
}

.child {
  /* Forces the child to take the parent's border */
  border: inherit; 
}
```

---

## 4. The "Initial" and "Unset" Keywords
If you want to break inheritance or reset a property, you have two other handy tools:

* **`initial`:** Sets the property to the default value defined by the CSS specification (e.g., `color: initial` usually turns text black).
* **`unset`:** This is a hybrid. It acts like `inherit` if the property naturally inherits, or like `initial` if it doesn’t.

---

## Common Gotcha: Buttons and Inputs
A very common source of frustration is that **Form Elements** (like `<button>`, `<input>`, and `<textarea>`) often do not inherit font properties from the body. Most browsers have "User Agent" styles that force these elements to use system fonts. 

**The Fix:**
To make sure your buttons look like the rest of your text, many developers use this snippet:
```css
button, input, select, textarea {
  font: inherit;
  color: inherit;
}
```

> **Prompt**: Please show me how to add theming support to a Django web application.

Theming support in a Django application generally falls into two categories: **Static Theming** (pre-defined modes like Light/Dark) and **Dynamic Theming** (user-customized colors or white-labeling).

In 2026, the industry standard is to leverage **CSS Variables (Custom Properties)** controlled by a data-attribute on the `<html>` or `<body>` tag. This allows for instant theme switching without a page reload.

---

## 1. The CSS Architecture
Instead of hardcoding colors, you define a "Theme Map" in your base stylesheet.

```css
/* base.css */
:root {
  /* Default Light Theme */
  --bg-color: #ffffff;
  --text-color: #1a1a1a;
  --primary-accent: #3b82f6;
}

[data-theme='dark'] {
  --bg-color: #121212;
  --text-color: #f5f5f5;
  --primary-accent: #60a5fa;
}

body {
  background-color: var(--bg-color);
  color: var(--text-color);
  transition: background-color 0.3s ease; /* Smooth transition */
}
```

---

## 2. Django Model for User Preferences
To persist the theme across sessions and devices, you should store the user's choice in a `Profile` model.

```python
# models.py
from django.db import models
from django.contrib.auth.models import User

class UserProfile(models.Model):
    THEME_CHOICES = [
        ('light', 'Light Mode'),
        ('dark', 'Dark Mode'),
        ('system', 'Follow System'),
    ]
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    theme_preference = models.CharField(
        max_length=10, 
        choices=THEME_CHOICES, 
        default='system'
    )
```

## 3. Injecting the Theme into Templates
To prevent the "Flash of Unstyled Content" (FOUC) where a dark-mode user briefly sees a white screen, you must apply the theme attribute as early as possible in the `base.html`.

### The Context Processor
First, ensure the theme is available to every template.
```python
# context_processors.py
def theme_processor(request):
    if request.user.is_authenticated:
        return {'active_theme': request.user.userprofile.theme_preference}
    return {'active_theme': 'system'}
```

### The Base Template

```html
<!DOCTYPE html>
<html lang="en" data-theme="{{ active_theme }}">
<head>
    <script>
        // Critical: Handle 'system' preference immediately
        const theme = "{{ active_theme }}";
        if (theme === 'system') {
            const systemTheme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
            document.documentElement.setAttribute('data-theme', systemTheme);
        }
    </script>
    <link rel="stylesheet" href="{% static 'css/base.css' %}">
</head>
<body>
    {% block content %}{% endblock %}
</body>
</html>
```

## 4. The Switching Logic (HTMX Approach)
In 2026, using **HTMX** is the most efficient way to toggle themes. It allows you to update the database and the UI simultaneously without a full page refresh.

**The Template Toggle:**
```html
<button hx-post="{% url 'toggle-theme' %}" 
        hx-swap="none"
        hx-on::after-request="toggleAttribute()">
    Toggle Dark Mode
</button>

<script>
    function toggleAttribute() {
        const html = document.documentElement;
        const current = html.getAttribute('data-theme');
        html.setAttribute('data-theme', current === 'dark' ? 'light' : 'dark');
    }
</script>
```

## 5. Advanced: Dynamic/White-Label Theming
If you are building a SaaS (like TallyApp) where users want their own branding, you can inject CSS variables directly into the style attribute of the body.

| Feature | Implementation | Use Case |
| :--- | :--- | :--- |
| **Static Toggle** | Data-attribute + CSS Variables | Standard Light/Dark mode |
| **User Colors** | Inline style injection: `style="--primary-accent: {{ user_color }};"` | Brand customization |
| **System Sync** | `prefers-color-scheme` Media Query | Accessibility/UX |



---

### Security & Performance Gotchas
* **Caching:** If using `@cache_page`, ensure you `Vary: Cookie` or include the theme in the cache key, otherwise a Light Mode user might see a cached Dark Mode page.
* **Content Security Policy (CSP):** If you allow users to pick hex codes for their themes, ensure your CSP allows `style-src 'unsafe-inline'` or use a nonce-based approach to prevent CSS injection attacks.

> **Prompt:** Please explain how to design prompts that generate higher-quality automated tests.

Designing prompts for high-quality automated tests in 2026 has moved beyond simple "write a test" commands. Modern best practices focus on **Context Engineering** and **Multi-Stage Reasoning** to avoid the "Single-Prompt" trap where AI produces shallow or brittle code.

---

## 1. The "Role-Task-Context" Framework

Professional prompt design starts with a structured framework that anchors the AI's "persona" and environmental knowledge.

* **Role:** Define the AI as a specific expert (e.g., "Senior SDET specializing in Playwright and robust locator strategies").
* **Task:** Use precise action verbs. Instead of "Check this," use "Generate a suite of 5 functional tests and 3 adversarial edge cases."
* **Context:** Provide the **Schema**, **User Story**, and **Framework constraints**.
* *Bad Context:* "Test the login page."
* *Good Context:* "Test the React login component located at `/src/auth/Login.tsx`. The API returns a 401 for invalid credentials and a JWT for success. Use Playwright's Page Object Model."



---

## 2. Advanced Prompting Patterns

Research shows that these three patterns significantly increase test reliability by forcing the AI to "think" before it "codes."

### A. Few-Shot Pattern (The "Gold Standard")

AI aligns 40% faster when shown what "good" looks like. Provide 2–3 examples of existing tests from your codebase to teach the AI your specific style, naming conventions, and mocking patterns.

> "Here are two examples of how we write unit tests for our Django models: `<example_1>`, `<example_2>`. Now, generate a similar test for the following new model..."

### B. Chain of Thought (CoT)

Ask the AI to explain its logic **before** writing the code. This prevents the "Hallucination of Correctness," where it writes code that looks right but asserts nothing meaningful.

> "First, list all possible edge cases for this function. Second, describe the expected behavior for each. Finally, output the code."

### C. Self-Critique & Revision Loop (SCR)

Prompt the AI to review its own draft. This is highly effective for catching syntax errors or missing boundary checks.

> "Write the test, then critique it for potential flakiness or missing assertions. Finally, provide the refined version."

---

## 3. Dealing with Modern AI Models (2026 Trends)

In 2026, the strategy differs based on whether you are using **Standard** or **Reasoning** models.

| Model Type | Prompting Strategy | Why? |
| --- | --- | --- |
| **Standard (e.g., GPT-5, Gemini 2.5)** | Use explicit "Step-by-Step" instructions. | They need guidance to stay on track. |
| **Reasoning (e.g., o-series, Claude Extended)** | Keep prompts **Lean and Atomic**. | They do internal "Chain of Thought" by default; over-prompting can confuse them. |

## 4. Summary of Prompt "Do's and Don'ts"

| Do ✅ | Don't ❌ |
| --- | --- |
| **Positive Framing:** "Use real data types." | **Negation:** "Don't use mock data." (The Pink Elephant problem). |
| **XML/Markdown Structure:** Use tags like `<context>` or `# Instructions`. | **Walls of Text:** Avoid long, unorganized paragraphs. |
| **Atomic Tasks:** One prompt = one specific test file or component. | **Mega-Prompts:** Asking for UI, API, and Documentation in one go. |

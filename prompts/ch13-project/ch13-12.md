> **Prompt**: How should I structure prompts when asking AI to explain unfamiliar code?

To effectively prompt an AI to explain unfamiliar code, you should move away from the "What does this do?" approach and toward a **Context-Aware, Structured Framework**. 

In 2026, models are capable of deep reasoning, but they need to know your current knowledge level and the specific "lens" through which you want the code analyzed (e.g., security, performance, or logic flow).

---

## 1. The "Code Anatomy" Prompt Structure
A high-quality explanation prompt follows the **R.C.T.O.** (Role, Context, Task, Output) model.

### Example Template:
> **Role:** Act as a Senior Software Architect.  
> **Context:** I am looking at a legacy Django repository. I understand Python well but am unfamiliar with this specific `services.py` file and how it interacts with the Celery worker.  
> **Task:** Explain the attached code. Focus on the data flow from the request to the background task.  
> **Output:** Provide a high-level summary, followed by a step-by-step logic breakdown, and finally, identify any potential "gotchas" or hidden side effects.

---

## 2. Advanced Prompting Techniques for Code
Using structured techniques ensures the AI doesn't just paraphrase the comments but actually deconstructs the logic.

* **Chain-of-Thought (CoT):** Add the phrase *"Think step-by-step through the execution flow"* to force the AI to trace variable states.
* **Analogical Reasoning:** Ask the AI to *"Explain this logic using a real-world analogy (e.g., a post office or a restaurant kitchen)"* to simplify complex architectural patterns.
* **The "Rubber Duck" Method:** Reverse the prompt: *"I will explain what I think this code does; tell me where my understanding is incorrect or incomplete."*



---

## 3. Specific "Lenses" for Deep Understanding
If the code is particularly dense, ask the AI to explain it through one of these specific frameworks:

| Lens | Prompt Phrase to Add | Best For... |
| :--- | :--- | :--- |
| **Data Flow** | "Trace the lifecycle of the `user_data` variable from input to DB." | Understanding state changes. |
| **Security** | "Audit this code for OWASP Top 10 vulnerabilities." | Catching hidden risks. |
| **Performance** | "Identify any O(n^2) operations or N+1 query problems." | Optimization. |
| **Educational** | "Explain this as if I am a Junior Developer." | Learning new syntax or libraries. |

---

## 4. Dealing with "Repo-Level" Context
In 2026, tools like **Claude Code** or **GitHub Copilot Workspace** allow you to ask about the whole folder, not just one file.

* **The "Seam" Prompt:** *"Where is the 'seam' between this module and the rest of the application? Which files import this, and why?"*
* **The "Intent" Prompt:** *"Based on the surrounding files and naming conventions, what was the likely original intent of this function, even if the current implementation is messy?"*



---

## 5. Summary Checklist for a "Perfect" Code Prompt
* [ ] **Provide the Code:** Paste the snippet or specify the file path.
* [ ] **Define Your Level:** (e.g., "I know React but not this state management library.")
* [ ] **Specify the Depth:** Do you want a 1-sentence summary or a line-by-line annotation?
* [ ] **Ask for Examples:** *"Show me a sample input and the resulting output for this function."*

How complex is the code you're currently trying to decipher—is it a single function or an entire interconnected module?

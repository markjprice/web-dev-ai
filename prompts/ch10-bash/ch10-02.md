> **Prompt**: Why is Bash still widely used when newer shells like `zsh` and `fish` exist?

Bash remains the industry standard not necessarily because it is the "best" or most feature-rich, but because of its position as the universal glue of the computing world. While **zsh** and **fish** offer better "quality of life" for human users, Bash excels in environments where humans aren't the ones typing.

---

## 1. The "Ubiquity" Factor (Everywhere by Default)

Bash is the default shell on almost every Linux distribution and is pre-installed on virtually every server, cloud instance (AWS, Azure, Google Cloud), and Docker container.

* **Reliability:** If you write a script in Bash, you can be 99% sure it will run on any server you touch.
* **Infrastructure as Code:** Most automation tools and CI/CD pipelines (like GitHub Actions or Jenkins) use Bash as their native language for executing tasks.

## 2. Legacy and Stability

Newer shells like **fish** often break "POSIX compliance" (the set of standards that govern how shells should behave).

* **Standardization:** Because Bash follows POSIX standards closely, code written 20 years ago still runs perfectly today.
* **Safety:** System administrators are hesitant to install "trendy" shells on mission-critical servers because a small syntax change in a newer shell could potentially break a script that manages backups or security.

## 3. The "Stack Overflow" Effect

Because Bash has been the king for decades, the ecosystem is massive.

* **Documentation:** If you have a problem, the solution is already on the internet.
* **Community:** Almost every tutorial, technical book, and Stack Overflow answer regarding the command line uses Bash syntax.

## 4. Shell vs. Scripting

There is a common distinction in the tech world: **interactive use** vs. **scripting.**

* **Interactive Use:** Many developers use **zsh** or **fish** for their daily work because they love the colorful autocompletion and themes (like "Oh My Zsh").
* **Scripting:** Even those same developers will usually write their scripts in **Bash** to ensure that their teammates can run the code without needing to install a new shell.

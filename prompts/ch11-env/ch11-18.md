> **Prompt:** Please demonstrate how to undo a bad commit safely.

Undoing a "bad commit" in Git depends entirely on two factors: **Have you pushed it to a shared server?** and **Do you want to keep the work you did?**

Here is the 2026 definitive guide to safe undoing.

---

## Scenario 1: The Commit is Local (Not Pushed)

If the mistake only exists on your computer, you can "rewind" time. This is the cleanest way to fix a typo or an accidental commit of the wrong files.

### A. Undo but keep the changes (`--soft`)

Use this if you committed too early and want to add more files or fix a typo.

```bash
git reset --soft HEAD~1

```

* **Result:** The commit disappears, but your work remains in the **Staging Area** (ready to be committed again).
* **Best for:** Adding a forgotten file to the previous commit.

### B. Undo and unstage the changes (`--mixed`)

This is the default mode.

```bash
git reset HEAD~1

```

* **Result:** The commit is gone, and your work is back in the **Working Directory** (unstaged).
* **Best for:** Realizing the whole approach was slightly wrong and you want to reorganize your changes.

### C. The "Nuclear" Option: Delete Everything (`--hard`)

**⚠️ Danger:** This permanently deletes your work.

```bash
git reset --hard HEAD~1

```

* **Result:** Both the commit and your code changes are deleted forever.

---

## Scenario 2: The Commit is Already Pushed

If you have pushed to a shared branch (like `main`), you **must not** use `reset`. Rewriting history that others have already pulled will cause "Integration Hell" for your team.

### The Professional Fix: `git revert`

Instead of deleting the bad commit, Git creates a **new** commit that is the mathematical "opposite."

```bash
git revert <commit-hash>
git push origin main

```

* **Why it's safe:** It keeps the history linear. If your bad commit added a line, the revert commit removes it. No one else's history is broken.
* **Audit Trail:** It provides a clear record that a mistake was made and then corrected.

---

## Scenario 3: I Just Need to Fix the Last Message

If the code is fine but the commit message is embarrassing or wrong:

```bash
git commit --amend -m "The new and improved message"

```

* **If already pushed:** You will need to use `git push --force-with-lease`.
* **Note:** In 2026, `--force-with-lease` is the standard over `--force` because it won't overwrite the remote if someone else has pushed new work in the meantime.

---

## Summary of "Undo" Commands

| Situation | Command | Safety Level |
| --- | --- | --- |
| **Local:** Fix message/files | `git commit --amend` | ✅ Safe (Local) |
| **Local:** Keep work, undo commit | `git reset --soft HEAD~1` | ✅ Safe (Local) |
| **Local:** Trash everything | `git reset --hard HEAD~1` | 🔴 **Destructive** |
| **Pushed:** Undo changes safely | `git revert <hash>` | ✅ **Safe (Team)** |
| **Pushed:** Delete from history | `git reset --hard` + `push --force-with-lease` | 🟠 **Risky** |

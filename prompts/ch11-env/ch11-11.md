> **Prompt:** Why is `COPY requirements.txt` done before `COPY ..`?

This practice is the single most effective way to speed up your Docker builds. It leverages **Docker’s Layer Caching** system to ensure you only reinstall dependencies when they actually change.

---

## 1. How Layer Caching Works

Every instruction in a `Dockerfile` (like `RUN`, `COPY`, or `FROM`) creates a **layer**.

* Before executing an instruction, Docker checks its cache to see if it has a layer for that exact command and those exact files.
* **The Golden Rule:** If a layer changes (invalidates), **every single layer after it** must be rebuilt from scratch, even if those later files haven't changed.

---

## 2. The "Slow" Way (Bad Optimization)

If you copy everything at once, any minor code change (like fixing a typo in a comment) will force a full reinstall of all your packages.

```dockerfile
FROM python:3.11
WORKDIR /app

# ❌ The BAD Way:
COPY . .                  # Any file change here breaks the cache
RUN pip install -r requirements.txt  # ⚡ This runs EVERY time you change code

```

In this scenario, if you change one line in `main.py`, Docker sees that `COPY . .` has different files, invalidates the cache, and forces `pip install` to run again—which can take minutes.

---

## 3. The "Fast" Way (Optimized)

By splitting the `COPY` command, you isolate the dependencies from the source code.

```dockerfile
FROM python:3.11
WORKDIR /app

# ✅ The OPTIMIZED Way:
COPY requirements.txt .    # This layer only breaks if requirements.txt changes
RUN pip install -r requirements.txt  # ⚡ This is now CACHED and instant!

COPY . .                   # Source code changes only break this final, fast layer

```

### Why this is faster:

1. **Initial Build:** Both ways take the same amount of time.
2. **Subsequent Builds (Code change only):** Docker checks `requirements.txt`. It hasn't changed? **Load from cache.** It checks the `pip install` layer. The previous layer was cached? **Load from cache.** 3. **Result:** Your build finishes in **seconds** instead of minutes because the "heavy lifting" of downloading and compiling packages is skipped.

---

## Summary of the Strategy

| Action | Frequency | Speed Impact | Cache Behavior |
| --- | --- | --- | --- |
| **Install OS Deps** | Rare | High | Place at the very top. |
| **Install Python/Node Deps** | Occasional | High | `COPY` manifest only + `RUN install`. |
| **Copy Source Code** | Constant | Low | Place at the very bottom. |

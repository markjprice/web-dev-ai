# Articles for *Web Dev with an AI Sidekick*

These longer articles expand on the teaching approach used in *Web Dev with an AI Sidekick*. They are intended for learners, teachers, colleges, bootcamps, librarians, parents, and anyone interested in learning web development with AI as a study partner rather than a shortcut.

- [The Four Jobs of an AI Sidekick](four-jobs.md)  
  A practical guide to using an AI coding assistant in four different ways while learning: teacher, coach, reviewer, and rubber duck. Includes examples of weak prompts and stronger prompts that keep the learner thinking.

- [AI Is a Sidekick, Not a Shortcut](ai-is-a-sidekick-not-a-shortcut.md)  
  Explains the main learning philosophy behind the book: AI can help beginners ask questions, debug, review, and verify their work, but learners still need to understand the code they use.

- [The AI Coding Assistant as an Intellectual Exoskeleton](the-ai-coding-assistant-as-an-intellectual-exoskeleton.md)  
  Uses the metaphor of an exoskeleton to describe how AI can help learners lift heavier conceptual loads without replacing their judgment, responsibility, or decision-making.

- [Web Development Is a Conversation Between Several Languages](conversation-between-languages.md)  
  Explains why the book teaches HTML, CSS, JavaScript, TypeScript, SQL, Python, Django, and Bash before the main project, and how these technologies work together in a web application.

- [We Should Stop Pretending Students Won’t Use AI While Learning to Code](we-should-stop-pretending.md)  
  Written for teachers, colleges, bootcamps, librarians, and course leaders. Discusses how to give beginners guardrails for using AI in ways that build understanding instead of hiding it.
